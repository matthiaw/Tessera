   - Implements `Connector` with `type()` returning `"unstructured-text"`
   - Inject `ExtractionService`, appropriate `TextChunker` (resolved by the connector config `chunk_strategy` field — default `ParagraphChunker`)
   - `capabilities()`: returns `new ConnectorCapabilities(false, false, false)` (read-only, no webhook, no write-back)
   - `poll(Clock clock, MappingDefinition mapping, ConnectorState state, TenantContext tenant)`:
     a. Read `folder_path` and `glob_pattern` (default `**/*.md`) from `mapping.raw()` JSONB (access the underlying parsed JSON from MappingDefinition — check how GenericRestPollerConnector accesses its config)
     b. Read `chunk_strategy` (default `"paragraph"`), `chunk_overlap_chars` (default `200`), `confidence_threshold` (default `0.7`), `provider` (must be `"anthropic"` — reject others)
     c. Scan the folder using `java.nio.file.Files.walk()` with the glob pattern (or `PathMatcher`)
     d. For each `.md` file: compute SHA-256 of file content. Compare against `state.extra().get("file_hashes")` (a `Map<String, String>` stored in ConnectorState JSONB). Skip unchanged files.
     e. For changed/new files: read content, chunk with configured `TextChunker`, extract entities via `ExtractionService.extract(chunk, tenant)`
     f. For each `ExtractionCandidate`: build a `CandidateMutation` with:
        - `targetTypeSlug = candidate.typeSlug()`
        - `targetNodeUuid = null` (create path; resolution happens in ConnectorRunner)
        - `properties = candidate.properties()` merged with `{ "name": candidate.name() }`
        - `sourceSystem = "unstructured-text:" + connectorId`
        - `connectorId = instance connector UUID from mapping`
        - `changeId = SHA-256 of (sourceDocumentId + ":" + candidate.typeSlug() + ":" + candidate.name())`
        - Provenance: `sourceDocumentId = SHA-256 of (connectorId + ":" + filePath + ":" + mtime)`, `sourceChunkRange = charOffset + ":" + charLength`, `extractorVersion = tessera version (from config or @Value)`, `llmModelId = from ExtractionService response metadata`, `extractionConfidence = candidate.confidence()`
     g. Build `nextState` with updated `file_hashes` map
     h. Return `PollResult(candidates, nextState, SyncOutcome.SUCCESS, dlqEntries)`
   - Handle extraction errors per chunk: catch `ExtractionException`, add to DLQ entries, continue with remaining chunks

<objective>
Build the entity resolution pipeline (three-tier: exact, embedding, fuzzy) and the extraction review queue (API + repository). The resolution service determines whether an extracted candidate matches an existing graph entity, and the review queue captures low-confidence candidates for operator decision.

Purpose: Entity resolution prevents duplicate graph nodes. The review queue protects graph integrity by routing uncertain extractions to human judgment before they enter the write funnel.

Output: EntityResolutionService, EmbeddingService, FuzzyNameMatcher, ExtractionReviewController, ExtractionReviewRepository, plus tests.
</objective>

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02.5-08 | Spoofing | ExtractionReviewController | mitigate | JWT authentication required; tenant from JWT claim, not URL path. All queries filter by model_id. |
| T-02.5-09 | Tampering | Review queue override | mitigate | Override only sets target_node_uuid (merge target); extracted_properties are not editable. Mutation still flows through GraphService.apply() with SHACL validation. |
| T-02.5-10 | Information Disclosure | Cross-tenant review queue | mitigate | findById AND findPending both filter by model_id. 404 on cross-tenant access (not 403). Tested in ExtractionReviewControllerIT. |
| T-02.5-11 | Elevation of Privilege | Review accept bypassing rules | mitigate | Accepted candidates go through full GraphService.apply() pipeline including rule engine and SHACL. Confidence is set to 1.0 but rules still fire. |
| T-02.5-12 | Tampering | False entity merge via embedding | accept | False merge rate < 5% per eval strategy. Default threshold 0.85 is conservative. Operator can review merges via review queue. |
</threat_model>

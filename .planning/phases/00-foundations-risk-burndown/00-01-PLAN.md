<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| developer → repo | Untrusted contributors clone and run `./mvnw verify`; build must not silently execute attacker-controlled code beyond standard Maven plugins. |
| Maven Central → build | Dependencies fetched at build time; supply-chain risk if BOMs drift. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-00-01 | Tampering | parent pom.xml dependency versions | mitigate | Pin every BOM and standalone dep with explicit version; `requireUpperBoundDeps` enforcer rule fails build on silent transitive upgrades. |
| T-00-02 | Tampering | Maven Wrapper jar download | mitigate | Pin `wrapperUrl` and `distributionUrl` to a specific Maven 3.9.9 release in `.mvn/wrapper/maven-wrapper.properties`; commit wrapper jar download checksum if Maven >= 3.9 supports it. |
| T-00-03 | Information disclosure | License headers missing on .java files | mitigate | `license-maven-plugin` `check` execution bound to `validate`; build fails before compile. |
| T-00-04 | Repudiation | Build hygiene drift between contributors | mitigate | Spotless + enforcer + license run from `validate` phase — no contributor can skip without `-Dskip*` overrides which are not advertised in CONTRIBUTING.md. |
| T-00-05 | Elevation of privilege | Plugin code execution at build time | accept | Trust Maven Central as the dependency source; ASVS L1 does not require SLSA attestation for OSS builds. Re-evaluate at Phase 5 hardening. |
</threat_model>

Outputs a stable text file with one line per query result (no timestamps, no order-sensitive output). Per D-06, this is a fixed query suite that runs identically before pg_dump and after pg_restore.

```bash
#!/usr/bin/env bash
# Fixed query suite — runs identically before pg_dump and after pg_restore.
# Per D-06: divergence in this output between pre/post is the failure signal.
# No whole-graph hash (too fragile) — explicit, sortable, line-stable results.
set -euo pipefail

OUT="${1:?usage: verify_queries.sh <output-file>}"
: "${PGHOST:=localhost}" "${PGPORT:=5432}" "${PGUSER:=tessera}" "${PGDATABASE:=tessera}"
export PGPASSWORD="${PGPASSWORD:-tessera}"

PSQL="psql -h $PGHOST -p $PGPORT -U $PGUSER -d $PGDATABASE -At -F $'\t' -X -q"

# Prime the session (LOAD 'age' is per-session)
$PSQL -c "LOAD 'age'; SET search_path = ag_catalog, '\"\$user\"', public;" >/dev/null

{
  echo "# Q1: count by label (CRIT-3 instrumentation)"
  $PSQL -c "SELECT * FROM cypher('tessera_bench', \$\$ MATCH (n) RETURN labels(n) AS l, count(*) AS c \$\$) AS (l agtype, c agtype)" \
    | sort

  echo "# Q2: count of edges"
  $PSQL -c "SELECT * FROM cypher('tessera_bench', \$\$ MATCH ()-[r:RELATES]->() RETURN count(r) AS c \$\$) AS (c agtype)"

  echo "# Q3: deterministic point-lookup — first 10 nodes by idx"
  $PSQL -c "SELECT * FROM cypher('tessera_bench', \$\$ MATCH (n:Person) RETURN n.idx AS i ORDER BY i ASC LIMIT 10 \$\$) AS (i agtype)"

  echo "# Q4: 2-hop reach from node idx=0"
  $PSQL -c "SELECT * FROM cypher('tessera_bench', \$\$ MATCH (n {idx: 0})-[:RELATES]->()-[:RELATES]->(m) RETURN count(DISTINCT m) AS c \$\$) AS (c agtype)"

  echo "# Q5: graph metadata"
  $PSQL -c "SELECT name FROM ag_catalog.ag_graph WHERE name = 'tessera_bench'"
} > "$OUT"

echo "verify_queries.sh: wrote $OUT ($(wc -l < "$OUT") lines)"
```

The output ordering uses explicit `ORDER BY` and `sort` to keep it stable. No timestamps anywhere.

Create `scripts/dump_restore_rehearsal.sh` (POSIX bash, +x). End-to-end flow per D-05/D-06/D-07:

```bash
#!/usr/bin/env bash
# FOUND-05: pg_dump / pg_restore rehearsal against a seeded AGE database.
# Per D-05: reuses the 100k benchmark dataset.
# Per D-06: pre/post fixed query suite; any divergence hard-fails (D-07).
# Per CRIT-1: AGE blocks pg_upgrade — this rehearsal IS the upgrade runbook.
set -euo pipefail

# Resolve digest from docker-compose.yml — single source of truth (D-09)
DIGEST=$(grep -oE 'apache/age@sha256:[a-f0-9]{64}' docker-compose.yml | head -1)
if [[ -z "$DIGEST" ]]; then
  echo "FAIL: could not extract apache/age sha256 digest from docker-compose.yml" >&2
  exit 1
fi
echo "Using image: $DIGEST"

WORKDIR=$(mktemp -d)
trap 'docker rm -f tessera-dump-src tessera-dump-dst >/dev/null 2>&1 || true; rm -rf "$WORKDIR"' EXIT

start_container() {
  local name="$1" port="$2"
  docker run -d --name "$name" \
    -e POSTGRES_USER=tessera -e POSTGRES_PASSWORD=tessera -e POSTGRES_DB=tessera \
    -p "$port:5432" "$DIGEST" >/dev/null
  for i in $(seq 1 30); do
    if docker exec "$name" pg_isready -U tessera >/dev/null 2>&1; then return 0; fi
    sleep 1
  done
  echo "FAIL: $name did not become ready" >&2; exit 1
}

# ------------------------------------------------------------------------
# 1. SOURCE: start container, run V1 init, seed 100k via Maven (reuse SeedGenerator).
# ------------------------------------------------------------------------
start_container tessera-dump-src 55432
docker exec -e PGPASSWORD=tessera tessera-dump-src \
  psql -U tessera -d tessera -v ON_ERROR_STOP=1 \
  -c "CREATE EXTENSION IF NOT EXISTS age" \
  -c "LOAD 'age'" \
  -c "SET search_path = ag_catalog, public" \
  -c "SELECT create_graph('tessera_bench')"

# Seed via the same SeedGenerator JMH harness uses (D-05). Run a minimal
# Java seed driver via Maven exec (single command, headless).
./mvnw -B -q -pl fabric-core exec:java \
  -Dexec.classpathScope=test \
  -Dexec.mainClass=dev.tessera.core.bench.SeedDriver \
  -Dseed.host=localhost -Dseed.port=55432 \
  -Dseed.db=tessera -Dseed.user=tessera -Dseed.password=tessera \
  -Dseed.count=100000

# Note: SeedDriver is a thin main() wrapper around SeedGenerator; if it does
# not yet exist, the executor must add it under fabric-core/src/test/java
# (kept in test scope so it isn't packaged). See acceptance criteria.

# ------------------------------------------------------------------------
# 2. Run the fixed query suite BEFORE the dump.
# ------------------------------------------------------------------------
PGHOST=localhost PGPORT=55432 PGUSER=tessera PGPASSWORD=tessera PGDATABASE=tessera \
  scripts/verify_queries.sh "$WORKDIR/pre.txt"

# ------------------------------------------------------------------------
# 3. pg_dump (custom format).
# ------------------------------------------------------------------------
docker exec -e PGPASSWORD=tessera tessera-dump-src \
  pg_dump -U tessera -d tessera -Fc -f /tmp/tessera.dump
docker cp tessera-dump-src:/tmp/tessera.dump "$WORKDIR/tessera.dump"

# ------------------------------------------------------------------------
# 4. DESTINATION: fresh container, restore, then re-init AGE session.
# ------------------------------------------------------------------------
start_container tessera-dump-dst 55433
# AGE extension must exist in the destination BEFORE restoring objects that
# depend on it. (CRIT-1 manifestation: vanilla pg_restore alone is not enough.)
docker exec -e PGPASSWORD=tessera tessera-dump-dst \
  psql -U tessera -d tessera -v ON_ERROR_STOP=1 \
  -c "CREATE EXTENSION IF NOT EXISTS age" \
  -c "LOAD 'age'" \
  -c "SET search_path = ag_catalog, public"

docker cp "$WORKDIR/tessera.dump" tessera-dump-dst:/tmp/tessera.dump
docker exec -e PGPASSWORD=tessera tessera-dump-dst \
  pg_restore -U tessera -d tessera --clean --if-exists --no-owner /tmp/tessera.dump

# ------------------------------------------------------------------------
# 5. Run the fixed query suite AFTER the restore.
# ------------------------------------------------------------------------
PGHOST=localhost PGPORT=55433 PGUSER=tessera PGPASSWORD=tessera PGDATABASE=tessera \
  scripts/verify_queries.sh "$WORKDIR/post.txt"

# ------------------------------------------------------------------------
# 6. Diff. D-07: any divergence hard-fails.
# ------------------------------------------------------------------------
if diff -u "$WORKDIR/pre.txt" "$WORKDIR/post.txt"; then
  echo "PASS: dump/restore rehearsal — pre and post query suites match (FOUND-05 / CRIT-1)"
  exit 0
else
  echo "FAIL: dump/restore divergence — see diff above (D-07 hard-fail)" >&2
  exit 1
fi
```

package dev.tessera.core.bench;

import java.sql.Connection;
import java.sql.DriverManager;

public final class SeedDriver {
    public static void main(String[] args) throws Exception {
        String host = System.getProperty("seed.host", "localhost");
        String port = System.getProperty("seed.port", "5432");
        String db   = System.getProperty("seed.db",   "tessera");
        String user = System.getProperty("seed.user", "tessera");
        String pw   = System.getProperty("seed.password", "tessera");
        int count   = Integer.parseInt(System.getProperty("seed.count", "100000"));
        try (Connection c = DriverManager.getConnection(
                "jdbc:postgresql://" + host + ":" + port + "/" + db, user, pw)) {
            try (var s = c.createStatement()) {
                s.execute("LOAD 'age'");
                s.execute("SET search_path = ag_catalog, public");
            }
            SeedGenerator.build(c, count);
        }
        System.out.println("seeded " + count + " nodes");
    }
    private SeedDriver() {}
}

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| GitHub Actions runner → repo | Workflows run with `contents: read` only — cannot push back to the repo. |
| GitHub Actions runner → Docker Hub | Pulls the digest-pinned `apache/age` image; sha256 prevents silent retag tampering. |
| nightly job → benchmark history | Results are uploaded as artifacts; baseline files in `.planning/benchmarks/` change only via PR. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-00-21 | Tampering | third-party GitHub Actions pinned by floating major (`@v4`) | accept | ASVS L1 does not require SHA-pinning of GH Actions; revisit at Phase 5 hardening with `pin-github-action`. Document in `.github/dependabot.yml` later. |
| T-00-22 | Elevation of privilege | workflow with excessive `permissions:` | mitigate | Both workflows declare `permissions: contents: read` explicitly (least privilege). |
| T-00-23 | Information disclosure | Postgres password leaking into CI logs | mitigate | Hardcoded local-only `tessera/tessera` credentials; no real secrets in Phase 0. Phase 2 will switch to Vault. |
| T-00-24 | Denial of service | dump_restore_rehearsal hangs on Docker pull | mitigate | `timeout-minutes: 60` on the rehearsal job; `start_container` loop bounded to 30s per container. |
| T-00-25 | Repudiation | nightly failure goes unnoticed | mitigate | D-07: hard-fail on divergence + GitHub Actions email-on-failure default + workflow status badge in README (Phase 1 follow-up). |
| T-00-26 | Tampering | dump file in mktemp tampered between dump and restore | accept | Single-runner ephemeral filesystem; no cross-process attacker model in CI. |
</threat_model>

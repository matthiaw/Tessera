**1. ListEntityTypesTool (MCP-02):**
- `toolName()` returns `"list_entity_types"`
- `toolDescription()` returns `"List all entity types available in the current tenant's schema. Returns type slugs, names, and descriptions."`
- `inputSchemaJson()` returns `"{\"type\":\"object\",\"properties\":{},\"required\":[]}"` (no parameters)
- `execute()`: Call `schemaRegistry.listNodeTypes(tenant)`. Map each `NodeTypeDescriptor` to a JSON object with fields `slug`, `name`, `description`, `schemaVersion`. Return `ToolResponse.ok(json)`.
- Inject: `SchemaRegistry`, `ObjectMapper`.

**2. DescribeTypeTool (MCP-02):**
- `toolName()` returns `"describe_type"`
- `toolDescription()` returns `"Describe a specific entity type including all properties, their data types, and connected edge types. Use list_entity_types first to discover available slugs."`
- `inputSchemaJson()` returns JSON schema with required `slug` (string) parameter.
- `execute()`: Extract `slug` from arguments. Call `schemaRegistry.loadFor(tenant, slug)`. If empty, return `ToolResponse.error("Type '" + slug + "' not found")`. Map the `NodeTypeDescriptor` to JSON including `slug`, `name`, `label`, `description`, `schemaVersion`, `properties` (list of {slug, dataType, required}). Also look up edge types: iterate known edge types via `schemaRegistry.findEdgeType()` for edges where sourceType or targetType matches — include `edgeTypes` array in response.
- Inject: `SchemaRegistry`, `ObjectMapper`.

**3. QueryEntitiesTool (MCP-03):**
- `toolName()` returns `"query_entities"`
- `toolDescription()` returns `"Query entities of a specific type with optional property filters and cursor-based pagination. Returns a page of entities and a next_cursor for pagination."`
- `inputSchemaJson()` returns JSON schema with required `type` (string), optional `filter` (object — property key-value pairs), optional `cursor` (string), optional `limit` (integer, default 20, max 100).
- `execute()`:
  1. Extract `type`, `filter`, `cursor`, `limit` from arguments. Default limit=20, cap at 100.
  2. If `cursor` present and non-blank: `CursorCodec.CursorPosition pos = CursorCodec.decode(cursor); afterSeq = pos.lastSeq();` else `afterSeq = 0`.
  3. Call `graphRepository.queryAllAfter(tenant, type, afterSeq, limit + 1)`.
  4. Apply in-memory filter if `filter` map is non-empty: for each entry in filter, only keep nodes where `properties.get(key).toString().equals(value.toString())`.
  5. Determine `hasMore = results.size() > limit`. Trim to `limit`.
  6. Build `nextCursor` via `CursorCodec.encode(tenant.modelId(), type, last.seq(), last.uuid())` if hasMore.
  7. Return JSON with `entities` array and `next_cursor` (null if no more).
- Inject: `GraphRepository`, `ObjectMapper`.

**4. GetEntityTool (MCP-04, D-B4):**
- `toolName()` returns `"get_entity"`
- `toolDescription()` returns `"Get a single entity by type and UUID, including connected entities up to the specified depth (0=entity only, 1=direct neighbors, 2=two hops, 3=three hops). Default depth is 1, maximum is 3."`
- `inputSchemaJson()` returns JSON schema with required `type` (string) and `id` (string/UUID), optional `depth` (integer, default 1, min 0, max 3).
- `execute()`:
  1. Extract `type`, `id`, `depth` from arguments. Default depth=1, clamp to [0,3].
  2. Call `graphRepository.findNode(tenant, type, UUID.fromString(id))`.
  3. If empty, return `ToolResponse.error("Entity not found")`.
  4. If depth > 0: execute a variable-length Cypher path query via `graphRepository.executeTenantCypher()`:
     ```
     MATCH path = (n {model_id: "modelId", uuid: "id"})-[*0..depth]-(m)
     WHERE m.model_id = "modelId"
     RETURN DISTINCT m
     ```
     Parse results into neighbor NodeState objects. Group by hop distance.
  5. Build JSON response: `entity` (the main node) + `neighbors` array (if depth > 0), each with their properties, type, uuid.
- Inject: `GraphRepository`, `ObjectMapper`.

**JSON serialization pattern for all tools:** Use a shared private method or utility to convert `NodeState` to a `Map<String, Object>` with keys: `uuid`, `type`, `properties`, `created_at`, `updated_at`. Use Jackson `ObjectMapper.writeValueAsString()` for final serialization.

**1. TraverseTool (MCP-05):**
- `@Component`, implements `ToolProvider`.
- `toolName()` returns `"traverse"`
- `toolDescription()` returns `"Execute a Cypher query against the tenant's graph. The query is automatically scoped to the current tenant — do not include model_id filters. Only read queries are allowed (no CREATE, DELETE, MERGE, SET, REMOVE, DROP). Example: MATCH (n:Person)-[:WORKS_AT]->(o:Organization) RETURN n, o LIMIT 10"`
- `inputSchemaJson()` returns JSON schema with required `query` (string) parameter.
- `execute()`:
  1. Extract `query` string from arguments.
  2. Validate non-null, non-blank. If blank, return `ToolResponse.error("query parameter is required")`.
  3. Call `graphRepository.executeTenantCypher(tenant, query)`.
  4. Serialize the `List<Map<String, Object>>` result to JSON.
  5. Return `ToolResponse.ok(json)`.
  6. Catch `IllegalArgumentException` from `executeTenantCypher` (mutation keyword blocked) and return `ToolResponse.error("Write operations are not permitted via traverse. Only read queries allowed.")`.
- Inject: `GraphRepository`, `ObjectMapper`.
- Does NOT override `isWriteTool()` — inherits default `false`.

**2. FindPathTool (MCP-06):**
- `@Component`, implements `ToolProvider`.
- `toolName()` returns `"find_path"`
- `toolDescription()` returns `"Find the shortest path between two entities by their UUIDs. Returns the ordered list of nodes along the path. Both nodes must belong to the current tenant."`
- `inputSchemaJson()` returns JSON schema with required `from` (string/UUID) and `to` (string/UUID) parameters.
- `execute()`:
  1. Extract `from` and `to` UUIDs from arguments. Parse with `UUID.fromString()`. Catch `IllegalArgumentException` and return `ToolResponse.error("Invalid UUID format")`.
  2. Call `graphRepository.findShortestPath(tenant, fromUuid, toUuid)`.
  3. If empty list, return `ToolResponse.ok("{\"path\": [], \"message\": \"No path found between the specified nodes\"}")`.
  4. Serialize the path as a JSON array of node objects (uuid, type, properties).
  5. Return `ToolResponse.ok(json)`.
- Inject: `GraphRepository`, `ObjectMapper`.
- **Read 03-00-SUMMARY.md** for the spike result on AGE shortestPath() agtype parsing. The tool delegates to `GraphRepository.findShortestPath()` which already uses the spike-confirmed pattern.

**3. GetStateAtTool (MCP-07):**
- `@Component`, implements `ToolProvider`.
- `toolName()` returns `"get_state_at"`
- `toolDescription()` returns `"Reconstruct the state of an entity at a specific point in time by replaying its event history. Returns the entity's properties as they were at the given timestamp. Useful for auditing, debugging, or understanding how data evolved."`
- `inputSchemaJson()` returns JSON schema with required `entity_id` (string/UUID) and `timestamp` (string/ISO-8601 instant) parameters.
- `execute()`:
  1. Extract `entity_id` and `timestamp` from arguments.
  2. Parse `UUID entityId = UUID.fromString((String) arguments.get("entity_id"))`. Catch `IllegalArgumentException` and return `ToolResponse.error("Invalid UUID format for entity_id")`.
  3. Parse `Instant at = Instant.parse((String) arguments.get("timestamp"))`. Catch `DateTimeParseException` and return `ToolResponse.error("Invalid timestamp format. Use ISO-8601 format, e.g. 2026-01-15T10:30:00Z")`.
  4. Call `eventLog.replayToState(tenant, entityId, at)`.
  5. If `Optional.empty()`, return `ToolResponse.ok("{\"state\": null, \"message\": \"No state found for entity at the specified timestamp\"}")`.
  6. Serialize the `Map<String, Object>` state to JSON with `entity_id`, `timestamp`, and `state` fields.
  7. Return `ToolResponse.ok(json)`.
- Inject: `EventLog`, `ObjectMapper`.

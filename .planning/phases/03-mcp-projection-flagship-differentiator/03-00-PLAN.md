Create `fabric-core/src/test/java/dev/tessera/core/graph/FindShortestPathSpikeIT.java`:

This is a **spike integration test** that validates RESEARCH.md Assumption A3: that AGE `shortestPath()` returns a parseable agtype path object.

Use the same Testcontainers + JDBC pattern as the existing `GraphRepositoryImplIT`. Do NOT use `@SpringBootTest` — use bare Testcontainers + JdbcTemplate for speed.

**Test setup:**
1. Start PostgreSQLContainer with pinned AGE image (same digest as CrossTenantLeakPropertyIT).
2. Connect via JDBC. Run AGE init: `CREATE EXTENSION IF NOT EXISTS age; LOAD 'age'; SET search_path = ag_catalog, "$user", public;`
3. Create graph: `SELECT create_graph('tessera_main');`
4. Seed 3 nodes (A->B->C chain) in tenant `model_id = "spike-tenant"`:
   ```sql
   SELECT * FROM cypher('tessera_main', $$
     CREATE (a:SpikeNode {model_id: "spike-tenant", uuid: "aaa-...", name: "A"})
     -[:CONNECTS_TO]->
     (b:SpikeNode {model_id: "spike-tenant", uuid: "bbb-...", name: "B"})
     -[:CONNECTS_TO]->
     (c:SpikeNode {model_id: "spike-tenant", uuid: "ccc-...", name: "C"})
     RETURN a, b, c
   $$) AS (a agtype, b agtype, c agtype);
   ```

**Test 1: shortestPath returns a parseable result:**
```sql
SELECT * FROM cypher('tessera_main', $$
  MATCH (a:SpikeNode {model_id: "spike-tenant", uuid: "aaa-..."}),
        (c:SpikeNode {model_id: "spike-tenant", uuid: "ccc-..."})
  MATCH path = shortestPath((a)-[*1..10]-(c))
  RETURN path
$$) AS (path agtype);
```
- Assert the query executes without error.
- Capture the raw agtype string from `rs.getString(1)`.
- Log the raw agtype string to stdout for inspection.
- Assert the result is non-null and non-empty.

**Test 2: nodes(path) extracts individual nodes:**
```sql
SELECT * FROM cypher('tessera_main', $$
  MATCH (a:SpikeNode {model_id: "spike-tenant", uuid: "aaa-..."}),
        (c:SpikeNode {model_id: "spike-tenant", uuid: "ccc-..."})
  MATCH path = shortestPath((a)-[*1..10]-(c))
  RETURN nodes(path)
$$) AS (nodes agtype);
```
- Assert returns an agtype array.
- Parse the array (it should be a JSON-like array of node objects).
- Assert 3 nodes extracted (A, B, C).
- Log each node's agtype representation.

**Test 3: Cross-tenant path isolation:**
- Seed a second tenant's node D: `model_id = "other-tenant"`.
- Create edge B->D (cross-tenant edge).
- Run shortestPath from A to C with model_id filter: `WHERE ALL(n IN nodes(path) WHERE n.model_id = "spike-tenant")`.
- Assert path does NOT include D.

**Spike outcome documentation:**
- If Tests 1-2 pass: Assumption A3 is CONFIRMED. `FindPathTool` can use `nodes(path)` to extract node data, then parse via existing agtype parsing patterns.
- If Test 1 fails (shortestPath unsupported): Fallback to variable-length MATCH: `MATCH p = (a)-[*1..10]-(c) RETURN nodes(p) LIMIT 1`. Document in SUMMARY.
- If Test 3 fails (WHERE ALL filtering does not work mid-traversal): Fall back to pre-validating source/target ownership before running path query, and post-filtering path results.

After running, add a comment at the top of the test file documenting the spike result (which pattern worked) for Plan 02 Task 2 to reference.

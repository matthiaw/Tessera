Create `fabric-projections/src/main/java/dev/tessera/projections/mcp/quota/AgentQuotaService.java`:
- `@Service` annotation.
- Inject `NamedParameterJdbcTemplate jdbc` and `McpAuditLog auditLog`.
- Private fields:
  - `ConcurrentHashMap<String, AtomicLong> hourlyCounters = new ConcurrentHashMap<>()` — key: `agentId|modelId`
  - `ConcurrentHashMap<String, Instant> hourlyWindowStarts = new ConcurrentHashMap<>()` — tracks window start
- SQL constant for quota lookup:
```java
private static final String LOAD_QUOTA = """
    SELECT writes_per_hour FROM mcp_agent_quotas
    WHERE agent_id = :agent_id AND model_id = :model_id::uuid
    """;
```
- Method `public void checkWriteQuota(TenantContext ctx, String agentId) throws QuotaExceededException`:
  1. Build key: `String key = agentId + "|" + ctx.modelId()`.
  2. Load quota limit from DB: query `mcp_agent_quotas` for `writes_per_hour`. If no row or `writes_per_hour == 0`, throw `new QuotaExceededException("Agent '" + agentId + "' has no write quota for model " + ctx.modelId())`.
  3. Check if current window expired (more than 1 hour since window start). If yes, reset counter to 0 and update window start.
  4. Increment counter: `long count = hourlyCounters.computeIfAbsent(key, k -> new AtomicLong(0)).incrementAndGet()`.
  5. If `count > limit`, throw `new QuotaExceededException("Write quota exceeded: " + count + "/" + limit + " writes/hour")`.

- Method `private int loadQuotaLimit(TenantContext ctx, String agentId)`:
  Execute LOAD_QUOTA SQL. Return 0 if no row found.

- Note on Pitfall 4 (restart counter initialization): On first access of a key, if the counter is newly created, check `auditLog.countForAgentSince(ctx, agentId, windowStart)` to warm the counter. This prevents quota bypass after restart. The `computeIfAbsent` creates the counter; immediately after, if the value is 0, call the warm-up query and set the counter to the returned value.

**1. McpAuditController (MCP-09 admin endpoint, D-C3):**

Create `fabric-projections/src/main/java/dev/tessera/projections/mcp/audit/McpAuditController.java`:
```java
@RestController
@RequestMapping("/admin/mcp")
public class McpAuditController {

    private final NamedParameterJdbcTemplate jdbc;

    public McpAuditController(NamedParameterJdbcTemplate jdbc) {
        this.jdbc = jdbc;
    }

    @GetMapping("/audit")
    public ResponseEntity<Map<String, Object>> getAuditLog(
            @RequestParam("model_id") UUID modelId,
            @RequestParam(value = "agent_id", required = false) String agentId,
            @RequestParam(value = "from", required = false) String from,
            @RequestParam(value = "to", required = false) String to,
            @RequestParam(value = "limit", defaultValue = "100") int limit,
            @AuthenticationPrincipal Jwt jwt) {
```
- Enforce tenant match: `jwt.getClaimAsString("tenant")` must equal `modelId.toString()`. Otherwise return 403.
- Cap limit at 500.
- Build dynamic SQL query against `mcp_audit_log`:
  - Base: `SELECT id, model_id, agent_id, tool_name, arguments, outcome, duration_ms, created_at FROM mcp_audit_log WHERE model_id = :model_id::uuid`
  - If `agentId` non-null: append `AND agent_id = :agent_id`
  - If `from` non-null: append `AND created_at >= :from::timestamptz` (parse ISO-8601)
  - If `to` non-null: append `AND created_at <= :to::timestamptz` (parse ISO-8601)
  - Append `ORDER BY created_at DESC LIMIT :limit`
- Execute query and return `ResponseEntity.ok(Map.of("entries", results, "count", results.size()))`.
- This endpoint is already protected by SecurityConfig's `/admin/**` rule requiring `ROLE_ADMIN` or `ROLE_TOKEN_ISSUER`.

The complete invokeTool flow becomes:
```
extractTenant() -> extractAgent() -> startTimer()
  -> [if writeTool: checkQuota()]
  -> tool.execute(ctx, agentId, params)
  -> ToolResponseWrapper.wrap(response.content())
  -> mcpAuditLog.record(ctx, agentId, toolName, params, outcome, durationMs)
  -> return CallToolResult
```

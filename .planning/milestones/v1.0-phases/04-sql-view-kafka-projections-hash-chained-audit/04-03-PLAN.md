<objective>
Replace the in-process outbox poller with Debezium 3.4 CDC for Kafka event fan-out, while retaining the poller as a development fallback. Add Docker Compose services for Kafka (KRaft) and Debezium Connect, plus an Actuator health indicator for replication slot monitoring.

Purpose: Kafka topics per (model_id, typeSlug) enable downstream consumers (BI pipelines, notification services, external systems) to subscribe to entity changes without polling REST endpoints. Debezium provides exactly-once semantics from the Postgres WAL without changing the write path.

Output: Docker Compose with Kafka + Debezium, Outbox Event Router connector config, conditionalized OutboxPoller, DebeziumSlotHealthIndicator, and tests.
</objective>

  1. Query replication slot lag: `SELECT pg_wal_lsn_diff(pg_current_wal_lsn(), confirmed_flush_lsn) FROM pg_replication_slots WHERE slot_name = :slot` with param `slot = "tessera_outbox_slot"`

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Debezium → PostgreSQL WAL | CDC reads WAL stream; slot must be authenticated |
| Kafka topics → downstream consumers | Events on topics contain tenant data; access must be controlled |
| Docker Compose credentials | Debezium connector JSON contains DB credentials (dev-only, not production) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-I3 | Information Disclosure | Kafka topics | mitigate | Topics are per-tenant per-type (`tessera.{model_id}.{type_slug}`). Production Kafka must use ACLs to restrict topic access. Docker Compose dev stack uses plaintext — acceptable for local dev. Document production Kafka ACL requirements. |
| T-04-T3 | Tampering | tessera-outbox.json | accept | Connector config contains dev DB credentials (tessera/tessera). Production deployments must use Vault-sourced credentials via Debezium's secret provider or environment variables. Accepted for dev-only Docker Compose. |
| T-04-D2 | Denial of Service | Replication slot WAL growth | mitigate | V26 sets `max_slot_wal_keep_size = 2GB` to cap WAL growth. DebeziumSlotHealthIndicator fires DOWN alert when lag exceeds threshold. Operator action required to resolve (restart Debezium, drop slot). |
| T-04-S3 | Spoofing | Debezium connector registration | accept | Connector init uses curl to Kafka Connect REST API (unauthenticated in dev). Production must secure Kafka Connect REST API with mTLS or basic auth. Accepted for local dev. |
</threat_model>

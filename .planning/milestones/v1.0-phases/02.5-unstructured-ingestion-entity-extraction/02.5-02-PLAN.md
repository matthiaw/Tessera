<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| ExtractionService → Anthropic API | LLM output is untrusted; may contain hallucinated entities, malformed JSON, or injection attempts |
| Schema Registry → ExtractionService | Schema drives LLM prompt; compromised schema could cause extraction of unintended types |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02.5-04 | Spoofing | ExtractionService LLM response | mitigate | Validate every ExtractionCandidate typeSlug against Schema Registry; reject unknown types. Source span verification against original text. |
| T-02.5-05 | Tampering | DynamicSchemaOutputConverter | mitigate | Parse JSON strictly; do not eval() or execute any code from LLM output. Jackson ObjectMapper with default secure settings. |
| T-02.5-06 | Denial of Service | ExtractionService retry loop | mitigate | Cap retries at 3 with exponential backoff (1s/2s/4s); total worst-case delay = 7s per chunk, bounded |
| T-02.5-07 | Information Disclosure | Anthropic API call | mitigate | API key loaded from Vault, never logged. Chunk text may contain tenant data — acceptable for extraction purpose; Anthropic DPA required for production PII. |
</threat_model>

## Decisions Made
- Added `fabric-rules` as a compile dependency of `fabric-connectors` -- ConnectorRunner needs to import EntityResolutionService, EmbeddingService, and ResolutionCandidate from fabric-rules. No circular dependency exists (fabric-rules does not depend on fabric-connectors).
- Extended MappingDefinition record from 6 to 12 fields with `@JsonIgnoreProperties(ignoreUnknown = true)` -- REST connectors pass null for the 6 unstructured fields; unstructured connectors pass null for the 6 REST fields. Jackson deserialization from JSONB handles both shapes.
- ConnectorRunner uses `@Autowired(required = false)` for EntityResolutionService, EmbeddingService, and ExtractionReviewRepository -- these beans may not exist in structured-only deployments where fabric-rules isn't loaded.
- Fixed glob matching: Java's `glob:**/*.md` PathMatcher does not match root-level files like `test.md`. Added dual-matcher pattern that also checks the stripped glob `*.md` when the pattern starts with `**/`.

<objective>
Fix the two circlead connector wiring bugs: (1) resolve Spring `${...}` placeholders in `sourceUrl` before `URI.create()` sees them, and (2) register circlead connector instances in the `connectors` DB table so `ConnectorRegistry.loadAll()` picks them up and `ConnectorScheduler.tick()` dispatches syncs. Purpose: Without these fixes, circlead connectors exist only as Spring beans but are invisible to the scheduler. `GenericRestPollerConnector.poll()` throws `IllegalArgumentException` from `URI.create()` because the placeholder is never resolved.

Output: Working `CircleadConnectorConfig` with placeholder resolution + DB upsert, application.yml config key, unit test proving resolution, integration test proving scheduler wiring.

@ExtendWith(MockitoExtension.class)
class CircleadSchedulerWiringIT {

    private final ApplicationContextRunner contextRunner = new ApplicationContextRunner()
        .withUserConfiguration(CircleadConnectorConfig.class)
        .withPropertyValues(
            "tessera.connectors.circlead.base-url=http://test-circlead:8080",
            "tessera.connectors.circlead.model-id=00000000-0000-0000-0000-000000000001",
            "tessera.connectors.circlead.credentials-ref=vault:test/token",

        .withBean(ObjectMapper.class, ObjectMapper::new)
        .withBean(NamedParameterJdbcTemplate.class, () -> mockJdbc)
        .withBean(DataSource.class, () -> mockDs);

    @Mock NamedParameterJdbcTemplate mockJdbc;
    @Mock DataSource mockDs;

    @Test
    void circlead_connector_config_registers_exactly_three_connectors() {
        contextRunner.run(ctx -> {
            // Verify the @PostConstruct fired and called jdbc.update() exactly 3 times
            verify(mockJdbc, times(3)).update(anyString(), anyMap());

            assertThatCode(() -> URI.create(role.sourceUrl())).doesNotThrowAnyException();
        });

}

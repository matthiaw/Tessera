    Structure (reuse patterns from dump_restore_rehearsal.sh):
    - `#!/usr/bin/env bash` + `set -euo pipefail`
    - Source the AGE image digest from docker-compose.yml (same grep pattern as existing script)
    - WORKDIR=$(mktemp -d), trap cleanup EXIT
    - Step numbering: 9 steps total

    Steps:
    1. Start source Postgres+AGE container (SRC) from pinned digest
    2. Wait for TCP readiness (same poll loop as existing script)
    3. Run Flyway migrate against SRC via `./mvnw -B -ntp -pl fabric-app flyway:migrate`
    4. Seed test data: INSERT into model_config (hash_chain_enabled=false, retention_days=30), INSERT a few graph_events rows for the test tenant
    5. pg_dump -Fc -n public from SRC container to $WORKDIR/tessera_dump.custom
    6. Start destination Postgres+AGE container (DST) on different port
    7. pg_restore into DST container
    8. Flyway validate against DST: `./mvnw -B -ntp -pl fabric-app flyway:validate` with DST port — verifies migration checksums match
    9. Smoke test: Query DST directly via psql to verify model_config.retention_days exists and graph_events rows survived restore. If any check fails, `echo "FAIL" && exit 1`.

    Note: The existing dump_restore_rehearsal.sh handles AGE-specific concerns (LOAD 'age', search_path). The DR drill does NOT start a full Tessera Spring Boot instance in CI (too heavy — requires all services). Instead it validates the DB layer: dump+restore+Flyway validate+data integrity query. This is sufficient for OPS-05 DR rehearsal.

    Make the script executable: `chmod +x scripts/dr_drill.sh`

    Add after the existing `benchmark-100k` job:

    ```yaml
    dr-drill:
      name: DR drill rehearsal
      runs-on: ubuntu-latest
      needs: verify
      if: github.event_name == 'push'
      timeout-minutes: 20
      steps:
        - uses: actions/checkout@v4
        - uses: actions/setup-java@v4
          with:
            distribution: corretto
            java-version: '21'
            cache: maven
        - name: Run DR drill
          run: scripts/dr_drill.sh
    ```

    Key decisions:
    - `needs: verify` — DR drill only runs after full Maven verify passes (all milestone-1 tests)
    - `if: github.event_name == 'push'` — skip on PRs to avoid Docker-in-Docker issues in some CI environments; push-to-main is the authority
    - `timeout-minutes: 20` — generous for dump/restore cycle with container startup
    - No separate Docker setup step needed — ubuntu-latest has Docker pre-installed

    Do NOT modify the existing `verify` or `benchmark-100k` jobs — they remain unchanged. The `verify` job already runs `./mvnw -B -ntp verify` which picks up all Phase 5 tests automatically once the test files exist.
